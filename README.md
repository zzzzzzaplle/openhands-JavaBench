# JavaBench

JavaBench 是一个面向 Agent 的 Java 编程基准测试项目，旨在评估和测试 AI Agent 在给定代码框架、UML 类图和需求描述的情况下，实现完整 Java 项目的能力。

## 项目结构

```
openhands-JavaBench/
├── pa19/                    # PA19 - 管道游戏 (Pipe Game)
│   └── input/
│       ├── 设置UML类图+需求/     # UML 类图和需求文档
│       └── 设置代码框架/          # 初始代码框架
├── pa20/                    # PA20 - 象棋游戏 (JesonMor)
│   └── input/
│       ├── 设置1代码框架/          # 初始代码框架
│       └── 设置2UML+需求/         # UML 类图和需求文档
├── pa21/                    # PA21 - 惯性游戏 (Inertia)
│   └── input/
│       ├── 设置1代码框架/          # 初始代码框架
│       └── 设置2UML+需求/         # UML 类图和需求文档
└── pa22/                    # PA22 - 推箱子游戏 (Sokoban)
    └── input/
        ├── 设置1代码框架/          # 初始代码框架
        └── 设置2UML+类图/          # UML 类图和需求文档
```

## 各作业简介

### PA19 - 管道游戏 (Pipe Game)

玩家需要在网格地图上铺设管道，让水流从起点流到终点。

**主要类：**
- `Pipe` - 管道基础类
- `Wall` - 墙壁类
- `TerminationCell` - 终止单元格
- `FillableCell` - 可填充单元格
- `Cell` - 单元格基类
- `PipeQueue` - 管道队列
- `CellStack` - 单元格栈（用于撤销）
- `DelayBar` - 延迟条
- `Map` - 游戏地图
- `Game` - 游戏主类

**关键概念：**
- 坐标系统 (Coordinate)
- 方向枚举 (Direction: UP, DOWN, LEFT, RIGHT)
- 管道形状 (PipeShape)

### PA20 - 象棋游戏 (JesonMor)

两人对战的棋盘游戏，包含骑士(Knight)和弓箭手(Archer)两种棋子。

**主要棋子：**
- `Knight` - 骑士（走 L 形，可被阻挡）
- `Archer` - 弓箭手（直线移动，吃子需有"炮架"）

**规则系统：**
- 移动验证规则管道
- 保护阶段（前 N 步不能吃子）
- 计分系统（基于曼哈顿距离）

### PA21 - 惯性游戏 (Inertia)

玩家选择一个方向后，角色会惯性滑动直到碰到墙壁或停止格。

**主要机制：**
- 惯性滑动
- 收集宝石 (Gem)
- 躲避地雷 (Mine)
- 生命系统
- 撤销系统

**计分公式：**
```
Score = InitialBoardSize + (Gems * 10) - (Moves * 1) - (Undoes * 2) - (Deaths * 4)
```

### PA22 - 推箱子游戏 (Sokoban)

经典推箱子游戏的多人变体，支持最多 26 名玩家。

**特色：**
- 每位玩家只能推动自己的箱子
- 支持撤销操作（可配置次数）
- 终端文本界面 (TUI)

**主要类：**
- `Position` - 坐标位置
- `GameMap` - 游戏地图
- `GameState` - 游戏状态
- `Player` - 玩家
- `Box` - 箱子
- `Action` - 动作基类
- `Move` - 移动动作
- `Undo` - 撤销动作

## 输入文件说明

每个作业的 `input` 目录下包含：

| 目录名 | 内容 |
|--------|------|
| 设置1代码框架/ | 已实现的类，提供方法签名和部分实现 |
| 设置2UML+需求/ | PlantUML 类图和详细需求描述 |
| 设置UML类图+需求/ | UML 类图和需求文档 |

## 使用方法

1. 阅读 `设置2UML+需求/` 目录下的需求文档和类图
2. 参考 `设置1代码框架/` 中的已有代码框架
3. 实现缺失的类和方法
4. 确保所有实现符合 UML 类图的设计

## 测试方式

每个作业通常包含：
- 代码框架自带的单元测试
- 地图文件用于游戏逻辑测试
